% ========== INTRODUCTION ==========
\section{Introduction}

\subsection{Contexte du projet}
L'École Nationale des Sciences Appliquées d'Al Hoceima (ENSAH) fait face à un besoin croissant de digitalisation des processus pédagogiques. Actuellement, la gestion des absences des étudiants est effectuée manuellement, ce qui entraîne :
\begin{itemize}
    \item Une perte de temps pour les enseignants et l'administration
    \item Un manque de traçabilité des absences
    \item Des difficultés à générer des statistiques fiables
    \item Une communication inefficace avec les étudiants
\end{itemize}

\subsection{Objectifs du projet}
Ce projet vise à développer une application web complète permettant :
\begin{itemize}
    \item La saisie électronique des absences par les enseignants
    \item La consultation des absences par les étudiants
    \item La gestion centralisée des comptes utilisateurs (Admin, Enseignant, Étudiant)
    \item La génération de statistiques et rapports
    \item La gestion des justifications et avertissements
\end{itemize}

% ========== TECHNOLOGIES ==========
\section{Technologies utilisées}

\subsection{Backend}
\begin{itemize}
    \item \textbf{PHP 8.0} : Langage serveur pour la logique métier et l'interaction avec la base de données
    \item \textbf{MySQL} : Système de gestion de base de données relationnelle
    \item \textbf{PDO} : Interface sécurisée pour les requêtes SQL (protection contre les injections)
\end{itemize}

\subsection{Frontend}
\begin{itemize}
    \item \textbf{HTML5/CSS3} : Structure et style des pages
    \item \textbf{Bootstrap 5} : Framework CSS responsive (interface moderne et adaptée mobile)
    \item \textbf{JavaScript} : Recherche en temps réel, confirmations, interactions dynamiques
    \item \textbf{Font Awesome} : Bibliothèque d'icônes vectorielles
\end{itemize}

\subsection{Environnement de développement}
\begin{itemize}
    \item \textbf{XAMPP} : Serveur local (Apache + MySQL + PHP)
    \item \textbf{Visual Studio Code} : Éditeur de code
    \item \textbf{phpMyAdmin} : Gestion graphique de la base de données
\end{itemize}

\subsection{Pourquoi ces technologies ?}
\begin{itemize}
    \item \textbf{PHP/MySQL} : Stack web mature, open-source, largement documentée
    \item \textbf{Bootstrap} : Accélère le développement responsive sans expertise CSS avancée
    \item \textbf{XAMPP} : Environnement unifié simple à installer sur toutes les plateformes
\end{itemize}

% ========== ARCHITECTURE ==========
\section{Architecture de l'application}

\subsection{Modèle MVC simplifié}
L'application suit une architecture de type MVC (Modèle-Vue-Contrôleur) :
\begin{itemize}
    \item \textbf{Modèle} : Fichiers PHP avec requêtes SQL (dans config/database.php)
    \item \textbf{Vue} : Templates HTML/Bootstrap (modules/*/liste.php, ajouter.php)
    \item \textbf{Contrôleur} : Logique métier dans chaque fichier PHP
\end{itemize}

\subsection{Structure des dossiers}
\begin{lstlisting}[language=bash, basicstyle=\ttfamily\small]
abscence/
├── config/            # Configuration base de donnees
├── includes/          # Composants reutilisables (header, footer, auth)
├── modules/           # Modules fonctionnels
│   ├── etudiants/     # CRUD etudiants
│   ├── comptes/       # Gestion des comptes utilisateurs
│   └── abscences/     # Gestion des absences
├── dashboard.php      # Accueil apres connexion
├── connexion.php      # Page d'authentification
└── logout.php         # Destruction session
\end{lstlisting}

\subsection{Base de données}
\begin{itemize}
    \item \textbf{utilisateurs} : id, login, mot\_de\_passe, role, enabled
    \item \textbf{etudiants} : id, nom, prenom, massar\_id, cin, email, niveau
    \item \textbf{absences} : id, etudiant\_id, enseignant\_id, date\_absence, type\_seance, etat
\end{itemize}

% ========== FONCTIONNALITÉS ==========
\section{Fonctionnalités détaillées}

\subsection{Module Authentification}
\begin{itemize}
    \item Connexion sécurisée par login/mot de passe (hachage bcrypt)
    \item Gestion de session PHP
    \item Déconnexion
    \item Protection des pages sensibles
\end{itemize}

\subsection{Module Gestion des étudiants}
\begin{itemize}
    \item Liste des étudiants avec recherche en temps réel (JavaScript)
    \item Ajouter, modifier, supprimer un étudiant
    \item Validation des données (email, CIN unique)
    \item Interface moderne avec statistiques
\end{itemize}

\subsection{Module Gestion des comptes}
\begin{itemize}
    \item Création de comptes (Admin, Enseignant, Étudiant)
    \item Génération automatique de mot de passe aléatoire
    \item Activation/Désactivation de comptes
    \item Réinitialisation de mot de passe
\end{itemize}

\subsection{Module Gestion des absences}
\begin{itemize}
    \item Saisie rapide par classe (sélection multiple)
    \item Liste des absences avec filtres
    \item Justification d'absence (avec motif)
    \item Annulation d'absence
    \item Statistiques (total, justifiées, non justifiées)
\end{itemize}

% ========== DÉFIS ==========
\section{Défis rencontrés et solutions}

\subsection{Problèmes techniques}
\begin{itemize}
    \item \textbf{Configuration XAMPP sur Mac} : Utilisation du port 8080 et des permissions de fichiers
    \item \textbf{Gestion des sessions} : Implémentation correcte de auth.php
    \item \textbf{Recherche en temps réel} : Utilisation de JavaScript addEventListener
    \item \textbf{Structure des dossiers} : Organisation MVC claire
\end{itemize}

\subsection{Améliorations possibles}
\begin{itemize}
    \item Ajout de notifications par email (PHPMailer)
    \item Export des données en CSV/PDF
    \item Graphiques statistiques avec Chart.js
    \item Interface dédiée pour les enseignants avec photos des étudiants
\end{itemize}
